# Security Policy

SkyShield is a security and compliance automation platform. We take vulnerabilities in this project seriously, and hold this repository to the same standard we expect it to enforce elsewhere.

## Supported Versions

SkyShield is currently in early foundational development (pre-release). Security support currently applies only to the `main` branch.

| Version | Supported |
|---------|-----------|
| `main`  | ✅ |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, report it privately by emailing:

**priyanshisaugat@gmail.com**

Please include:

- A description of the vulnerability and its potential impact.
- Steps to reproduce, or a proof of concept if available.
- Any suggested remediation, if known.

## Response Expectations

- You will receive an acknowledgment of your report within a reasonable timeframe.
- We will investigate and, if confirmed, work on a fix and coordinate disclosure timing with you.
- Credit will be given to the reporter unless anonymity is requested.

## Scope

As SkyShield grows to include infrastructure-as-code scanning, container scanning, secret detection, and AWS integrations, this policy will be expanded to cover each component explicitly. Until then, this policy applies to the repository as a whole.
